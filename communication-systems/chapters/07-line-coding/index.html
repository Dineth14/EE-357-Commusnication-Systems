Overview

Line coding converts a sequence of binary data into a waveform suitable for baseband transmission. Key properties to consider:

  • DC balance: Zero DC component prevents baseline wander in AC-coupled channels.
  • Self-clocking: Frequent transitions allow clock recovery at the receiver.
  • Bandwidth efficiency: Minimum bandwidth for given data rate.
  • Error detection: Some codes have built-in error detection capability.

NRZ Codes

NRZ-L (Non-Return-to-Zero Level)

1 → +V, 0 → −V. Simple but has DC component and no self-clocking.

NRZ-I (Non-Return-to-Zero Inverted)

1 → transition, 0 → no transition. Differential encoding provides immunity to polarity inversion.

$$\text{NRZ Bandwidth:}\quad B_{min} = \frac{R_b}{2} = \frac{1}{2T_b}$$

Return-to-Zero (RZ)

Signal returns to zero in the middle of each bit interval. 1 → +V then 0, 0 → −V then 0.

$$\text{RZ Bandwidth:}\quad B_{min} = R_b = \frac{1}{T_b}$$

RZ requires twice the bandwidth of NRZ but provides self-clocking.

Manchester Coding

Also called biphase-L. Each bit has a mid-bit transition: 1 → high-to-low, 0 → low-to-high. Used in 10 Mbps Ethernet.

$$\text{Manchester Bandwidth:}\quad B_{min} = R_b = \frac{1}{T_b}$$

Advantages: guaranteed self-clocking, zero DC component, error detection (missing transitions).

Differential Manchester: Transition at mid-bit always present (for clocking). Transition at bit boundary encodes data (no transition = 1, transition = 0).

AMI & HDB3

Alternate Mark Inversion (AMI): 0 → 0V, 1 → alternating +V/−V. Zero DC, but long runs of zeros lose clock sync.

HDB3 (High-Density Bipolar 3): Replaces runs of 4+ zeros with a violation pattern to maintain clock sync while preserving zero DC property.

Power Spectral Density

$$S_{NRZ\text{-}L}(f) = T_b\,\text{sinc}^2(fT_b)$$
$$S_{Manchester}(f) = T_b\,\text{sinc}^2\!\left(\frac{fT_b}{2}\right)\sin^2\!\left(\frac{\pi f T_b}{2}\right)$$
$$S_{AMI}(f) = T_b\,\text{sinc}^2(fT_b)\sin^2(\pi fT_b)$$

Note: NRZ-L has energy at DC. Manchester and AMI have nulls at DC → no baseline wander.

Eye Diagrams

An eye diagram is formed by superimposing successive symbol-period segments of the received waveform. Key measurements:

  • Eye opening (vertical): Noise margin — larger is better
  • Eye width (horizontal): Timing margin — wider allows more jitter tolerance
  • Best sampling instant: Where eye is most open (maximum separation)
  • Zero crossings spread: Timing jitter indicator
